Log:
Feature Request #4288 ("Preserve scrollbar positions") realized.
The main intention is achieved: usability of long result lists and recipient lists (in the property panel) is reasonably improved. However, one bug remains: the scrollbar is always adjusted in the way the caret is inside the viewport. Thus, a scrollbar position which sets the caret out of sight is not accurately restored. Note: also in disabled fields the caret position can be set by clicking.
